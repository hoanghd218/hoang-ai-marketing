# 5 bước setup AI agent tự động bảo trì code

Bạn vừa ship code xong. Ai sẽ review cho bạn?

Nếu bạn là solopreneur hay làm việc một mình, câu trả lời thường là: không ai cả. Code lên production, cầu trời phù hộ.

Nhưng bây giờ bạn có thể setup một AI agent tự động làm việc này. Chỉ 5 bước.

Bước 1: Chọn trigger (khi nào agent thức dậy)
Đặt lịch chạy hàng ngày, hoặc bật mỗi khi có code mới push lên GitHub, hoặc khi có tin nhắn trên Slack. Bạn quyết định khi nào agent cần làm việc.

Bước 2: Chọn repo (agent làm việc ở đâu)
Chỉ định repo nào agent được phép truy cập. Nó sẽ chạy trong môi trường cloud sandbox -- cách ly hoàn toàn, không ảnh hưởng máy bạn.

Bước 3: Viết prompt (agent làm gì)
Đây là phần quan trọng nhất. Viết rõ ràng: "Review code mới, kiểm tra bảo mật, báo cáo lỗi." Prompt càng cụ thể, kết quả càng tốt.

Bước 4: Chọn model (AI nào chạy)
Có thể dùng Claude Opus, Sonnet, GPT, hay Codex. Mỗi model có thế mạnh riêng.

Bước 5: Kết nối MCP tools (công cụ bổ sung)
Thêm các công cụ AI cần dùng: gửi thông báo Slack, tạo Pull Request, đọc tài liệu. MCP (cầu nối giúp AI nói chuyện với các app khác) mở rộng khả năng của agent.

Done. Bật lên. Agent tự động chạy mỗi khi có trigger.

Kết quả: Code được review tự động. Lỗi bảo mật được phát hiện sớm. Bạn tập trung vào việc quan trọng hơn.

Setup một lần -- chạy mãi. Đó là lợi thế của người biết dùng AI đúng cách.
