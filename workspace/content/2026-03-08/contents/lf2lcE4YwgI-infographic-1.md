# 4 cách chạy tác vụ tự động với Claude Code -- bạn đang dùng cách nào?

**Slide 1 - Hook:**
Claude Code có 4 cách tự động hoá tác vụ.
Nhưng 90% người chỉ biết dùng 1.

**Slide 2 - Agent Loops (CLI):**
Cách dùng: /loop + thời gian + lệnh
Ưu điểm: Đơn giản, nhanh, tiện
Nhược điểm: Mất khi đóng terminal, max 3 ngày
Phù hợp: Tác vụ ngắn hạn trong ngày (kiểm tra deploy, chạy test)

**Slide 3 - Desktop Scheduler:**
Cách dùng: Claude Code Desktop > Scheduled > New Task
Ưu điểm: Không hết hạn, sống sót restart, không chặn công việc
Nhược điểm: Cần giữ Desktop mở, tạo session mới mỗi lần
Phù hợp: Tác vụ dài hạn, lặp lại hàng ngày (báo cáo sáng, tổng hợp Slack)

**Slide 4 - GitHub Actions:**
Cách dùng: Cấu hình workflow trong GitHub
Ưu điểm: Không cần bật máy tính, chạy trên cloud
Nhược điểm: Chỉ cho tác vụ liên quan GitHub (PR, code review)
Phù hợp: Tự động fix lỗi code, review PR, chạy CI/CD

**Slide 5 - Remote Control:**
Cách dùng: Claude Code Remote
Ưu điểm: Điều khiển Claude Code từ thiết bị khác
Nhược điểm: Vẫn session-based, tắt khi đóng
Phù hợp: Làm việc từ xa trên dự án cụ thể

**Slide 6 - CTA:**
Biết dùng công cụ nào, đúng lúc nào -- đó là siêu năng lực của người dùng AI.
